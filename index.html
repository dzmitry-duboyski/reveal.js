Monolith problems:

  • High complexity
  • Slow development

  • Difficult to scale
  • Hard to test
  • Dependence on a gradually aging technology stack

idea

Microservices is a design style that breaks down a product into separate services with different independent functions.

Microservice is a standalone, independently deployable software component that implements useful functions.

microservice development

One team develops and maintains one microservice.

Pros of microservice architecture:

  • Microservices are deployed independently of each other
  • Microservices are not large and easy to maintain
  • Independent scaling of microservices
  • Autonomy of development teams
  • An opportunity to experiment and introduce new technologies
  • Error isolation
Disadvantages of microservice architecture:
  • Itâ€™s hard to find the right set of microservices
  • The complexity of distributed systems makes development, testing, and deployment difficult
  • Deploying features spanning multiple microservices requires careful coordination
  • The decision about when to switch to microservice architecture is nontrivial

Docker

Docker is software for automating the deployment and management of applications in containerized environments.

Docker Allows you to "pack" the application with all its surroundings and dependencies into a container that can be transferred to any system, and also provides an environment for managing containers.

Docker consists of the following components:

  • Docker engine
  • Docker hub
  • Docker swarm
  • Docker compose
  • Docker kinematic
  • Docker machine

What is a container ??

A container is a packaged application with all its dependencies and the environment.

Pros of using a container:

  • Simple launch of the container
  • Guarantee that our application will run on a different host
  • High performance and speed
